## Overview

The Graph Overlay provides a lightweight graph abstraction on top of SochDB's key-value storage. It's designed for **agent memory** use cases: tracking conversations, entities, actions, and their relationships without the complexity of a full graph database.

<Note>
**Why Graph Overlay?** Traditional graph databases are heavy (Neo4j, ArangoDB). SochDB's Graph Overlay gives you graph operations (BFS, DFS, shortest path) on top of your existing embedded database with O(1) node/edge operations.
</Note>

## Key Features

- **Node & Edge Storage**: Store entities and relationships with properties
- **Graph Traversal**: BFS, DFS, shortest path with configurable depth limits
- **Neighbor Queries**: Find connected nodes with edge type filtering
- **Namespace Isolation**: Separate graphs per agent/tenant
- **No External Dependencies**: Runs in your existing SochDB instance

## Storage Model

The graph is stored using SochDB's path-based storage:

```
_graph/{namespace}/nodes/{node_id} → {type, properties}
_graph/{namespace}/edges/{from}/{edge_type}/{to} → {properties}
_graph/{namespace}/index/{edge_type}/{to}/{from} → {from_id}  (reverse index)
```

This enables:
- **O(1) node lookup**: Direct path access
- **O(degree) neighbor queries**: Prefix scan on edges
- **O(1) edge operations**: No graph rebuild needed

## Quick Start

### Python

<CodeGroup>
```python Agent Conversation Memory
from sochdb import Database, GraphOverlay

db = Database.open("./agent_db")
graph = GraphOverlay(db, namespace="agent_001")

# Track a conversation as a graph
graph.add_node("msg_1", node_type="message", properties={
    "role": "user",
    "content": "What's the weather in San Francisco?",
    "timestamp": "2024-03-15T10:00:00Z"
})

graph.add_node("msg_2", node_type="message", properties={
    "role": "assistant",
    "content": "Let me check the weather for you."
})

graph.add_node("tool_1", node_type="tool_call", properties={
    "tool": "weather_api",
    "location": "San Francisco",
    "result": "Sunny, 72°F"
})

graph.add_node("msg_3", node_type="message", properties={
    "role": "assistant",
    "content": "It's sunny and 72°F in San Francisco."
})

# Link the conversation flow
graph.add_edge("msg_1", "msg_2", edge_type="triggers")
graph.add_edge("msg_2", "tool_1", edge_type="invokes")
graph.add_edge("tool_1", "msg_3", edge_type="provides_context")

# Traverse the conversation
path = graph.bfs("msg_1", max_depth=10)
print(f"Conversation flow: {' → '.join(path)}")
# Output: msg_1 → msg_2 → tool_1 → msg_3
```

```python Entity Relationship Tracking
db = Database.open("./crm_db")

    "name": "Alice Smith",
    "email": "alice@example.com",

    "name": "Acme Corp",
    "industry": "Technology"
})

graph.add_node("project_ai", node_type="project", properties={
    "name": "AI Initiative",
    "budget": 500000
})

# Add relationships
graph.add_edge("person_alice", "company_acme", edge_type="works_at", properties={
    "start_date": "2020-01-15",
    "department": "Engineering"
})

graph.add_edge("person_alice", "project_ai", edge_type="leads", properties={
    "role": "Tech Lead"
})

graph.add_edge("company_acme", "project_ai", edge_type="sponsors")

neighbors = graph.get_neighbors("person_alice", direction="both")

    node = graph.get_node(neighbor.node_id)

# Alice → works_at → Acme Corp
# Alice → leads → AI Initiative
```

### Rust

```rust
use sochdb::graph::{GraphOverlay, GraphNode, GraphEdge};
use sochdb::Connection;
use std::collections::HashMap;

let conn = Connection::open("./agent_memory")?;
let graph = GraphOverlay::new(conn, "agent_001");

// Create nodes
let mut props = HashMap::new();
props.insert("role".to_string(), serde_json::json!("user"));
props.insert("content".to_string(), serde_json::json!("Hello"));

graph.add_node("msg_1", "message", Some(props))?;

// Create edges
graph.add_edge("msg_1", "replied_to", "msg_2", None)?;

// Traverse
let path = graph.shortest_path("msg_1", "msg_5", 10, None)?;
println!("Path: {:?}", path);
```

## Graph Traversal

### Breadth-First Search (BFS)

Explore nodes level by level (nearest neighbors first):

```python
# Find all nodes reachable from msg_1 within 3 hops
nodes = graph.bfs(
    start_id="msg_1",
    max_depth=3,
    edge_types=None,  # All edge types
    node_types=None   # All node types
)
print(f"Reachable nodes: {nodes}")
# Output: ['msg_1', 'msg_2', 'tool_1', 'msg_3', 'msg_4']

# Filter by edge type
conversation_flow = graph.bfs(
    start_id="msg_1",
    max_depth=10,
    edge_types=["triggers", "provides_context"],
    node_types=None
)

# Filter by node type
messages_only = graph.bfs(
    start_id="msg_1",
    max_depth=10,
    edge_types=None,
    node_types=["message"]
)
```

### Depth-First Search (DFS)

Explore as far as possible along each branch before backtracking:

```python
# Find all dependencies (action pipeline)
actions = graph.dfs(
    start_id="action_1",
    max_depth=10,
    edge_types=["feeds_into", "depends_on"],
    node_types=["action"]
)
print(f"Action pipeline: {actions}")
# Output: ['action_1', 'action_2', 'action_3', 'action_4']
```

### Shortest Path

Find the shortest path between two nodes:

```python
path = graph.shortest_path(
    from_id="user_alice",
    to_id="project_ai",
    max_depth=5,
    edge_types=None  # Consider all edge types
)

if path:
    print(f"Connection: {' → '.join(path)}")
    # Output: user_alice → company_acme → project_ai
else:
    print("No path found")
```

### Neighbor Queries

Find all directly connected nodes:

```python
from sochdb import EdgeDirection

# Outgoing neighbors only
outgoing = graph.get_neighbors(
    "user_alice",
    edge_types=None,
    direction=EdgeDirection.OUTGOING
)

# Incoming neighbors only
incoming = graph.get_neighbors(
    "company_acme",
    edge_types=["works_at"],
    direction=EdgeDirection.INCOMING
)

# Both directions
all_neighbors = graph.get_neighbors(
    "user_alice",
    edge_types=None,
    direction=EdgeDirection.BOTH
)

for neighbor in all_neighbors:
    node = graph.get_node(neighbor.node_id)
    edge = neighbor.edge
    print(f"{edge.edge_type}: {node.properties['name']}")
```

## Performance

| Operation | Time Complexity | Notes |
|-----------|----------------|-------|
| Add node | O(1) | Direct key write |
| Get node | O(1) | Direct key read |
| Add edge | O(1) | Two writes (edge + reverse index) |
| Get edges | O(degree) | Prefix scan |
| BFS/DFS | O(V + E) | Standard graph traversal |
| Shortest path | O(V + E) | BFS-based |

<Note>
**Scalability**: Graph Overlay is optimized for agent memory (hundreds to thousands of nodes per agent). For very large graphs (millions of nodes), consider a dedicated graph database.
</Note>

## Best Practices

<CardGroup cols={2}>
  <Card title="Use Namespaces" icon="layer-group">
    Isolate graphs per agent/tenant:
    ```python
    # Per-agent memory
    agent_graph = GraphOverlay(db, f"agent_{agent_id}")
    
    # Per-user knowledge
    user_graph = GraphOverlay(db, f"user_{user_id}")
    ```
  </Card>
  
  <Card title="Meaningful Node IDs" icon="tag">
    Use descriptive node IDs for debugging:
    ```python
    # Good
    graph.add_node("msg_2024-03-15_001", "message", ...)
    graph.add_node("user_alice_smith", "person", ...)
    
    # Avoid
    graph.add_node("n_1", "message", ...)  # Unclear
    ```
  </Card>
  
  <Card title="Typed Edges" icon="arrow-right">
    Use specific edge types for clarity:
    ```python
    # Good
    graph.add_edge(msg1, msg2, "triggers")
    graph.add_edge(msg2, tool, "invokes")
    graph.add_edge(tool, msg3, "provides_context")
    
    # Avoid
    graph.add_edge(msg1, msg2, "related")  # Vague
    ```
  </Card>
  
  <Card title="Limit Traversal Depth" icon="layer-group">
    Always set `max_depth` to prevent runaway traversals:
    ```python
    # Good
    nodes = graph.bfs(start, max_depth=10)
    
    # Dangerous (could traverse entire graph)
    nodes = graph.bfs(start, max_depth=999999)
    ```
  </Card>
</CardGroup>

## Integration with Vector Search

Combine graph structure with semantic search:

```python
from sochdb import Database, GraphOverlay

db = Database.open("./agent_db")
graph = GraphOverlay(db, "agent_memory")
collection = db.namespace("agent_memory").collection("messages")

# 1. Store message in vector collection
collection.insert(
    id="msg_42",
    vector=embedding,
    metadata={"content": "How do I optimize queries?"}
)

# 2. Store message in graph
graph.add_node("msg_42", "message", {"content": "How do I optimize queries?"})

# 3. Link to conversation
graph.add_edge("msg_41", "msg_42", "followed_by")

# 4. Hybrid retrieval: semantic search + graph context
results = collection.search(query_embedding, k=5)
top_msg_id = results[0].id

# Get conversation context via graph
context_ids = graph.bfs(top_msg_id, max_depth=3, node_types=["message"])
context_msgs = [graph.get_node(nid) for nid in context_ids]
```
