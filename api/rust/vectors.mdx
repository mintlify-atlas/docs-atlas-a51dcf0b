## VectorCollection

Scale-aware vector collection that automatically selects the optimal backend based on collection size.

### Backend Selection

| Scale | Backend | Memory/Vector | Search Time |
|-------|---------|---------------|-------------|
| {"<"}100K | HNSW | 1536 bytes | {"<"}1ms |
| 100K+ | Vamana+PQ | 48 bytes | 1-5ms |

### Creating Collections

```rust
use sochdb::prelude::*;

let conn = Connection::open("./data")?;

// Create with dimension
let mut coll = VectorCollection::create(&conn, "embeddings", 768)?;

// Or open existing
let coll = VectorCollection::open(&conn, "embeddings")?;
```

### add()

```rust

    vec![0.1; 768],
    vec![0.2; 768],

```

**Example:**

```rust

```

## Searching Vectors

### search()

Find k-nearest neighbors:

```rust
pub fn search(&self, query: &[f32], k: usize) -> Result<Vec<SearchResult>>
```

**Example:**

```rust
let query = vec![0.15; 768];
let results = coll.search(&query, 10)?;

for result in results {
    println!("{}: distance={}", result.id, result.distance);
}
```

```rust

```

**Example with metadata:**

```rust
for result in results {

    println!("Distance: {:.4}", result.distance);

        println!("Metadata: {:?}", meta);
    }

```

## Product Quantization (PQ)

Memory-efficient vector compression for large collections.

### Compression Ratio

For 768-dimensional vectors:
- Original: 768 × 4 = 3,072 bytes
- PQ (M=96, k=256): 96 bytes
- **32x compression**

### train_pq()

Train the Product Quantizer on existing vectors:

```rust
pub fn train_pq(&mut self) -> Result<()>
```

<Note>
Requires at least 1,000 vectors for good results.
</Note>

**Example:**

```rust
// Add training vectors
for i in 0..1500 {
    let vec = generate_embedding();  // Your embedding generation
    coll.add_one(&format!("doc{}", i), vec)?;
}

// Train PQ for compression
coll.train_pq()?;

if coll.is_pq_trained() {
    println!("Compression ratio: {:.1}x", coll.compression_ratio().unwrap());
}
```

### ProductQuantizer

Low-level PQ implementation:

```rust
pub struct ProductQuantizer {
    m: usize,          // Number of subspaces
    k: usize,          // Centroids per subspace
    d_sub: usize,      // Dimensions per subspace
    dimension: usize,  // Total dimension
    // ...
}
```

**Creating a quantizer:**

```rust
// Default parameters (optimized for LLM embeddings)
let mut pq = ProductQuantizer::new_default(768);

// Custom parameters
let mut pq = ProductQuantizer::new(
    768,   // dimension
    96,    // subspaces
    256    // centroids per subspace
);

// Train on vectors
pq.train(&training_vectors, 20)?;  // 20 k-means iterations

// Encode vector to PQ codes (96 bytes)
let code = pq.encode(&vector);

// Decode back to approximate vector
let approx = pq.decode(&code);
```

## Distance Metrics

Currently supports Euclidean distance (L2):

```rust
fn euclidean_distance(a: &[f32], b: &[f32]) -> f32
```

Uses SIMD-accelerated implementation:
- **~8x faster** than scalar on AVX2
- 768-dim: ~288 cycles vs 2304 cycles

## Gradual Migration

When a collection exceeds 100K vectors, SochDB automatically migrates from HNSW to Vamana+PQ:

### Migration Process

1. **Hybrid Mode**: New vectors written to both backends
2. **Batch Migration**: Old vectors migrated in chunks (default: 1000)
3. **Intelligent Routing**: Queries use optimal backend during migration
4. **Finalization**: HNSW deallocated when complete

### migrate_batch()

Manually trigger batch migration:

```rust
pub fn migrate_batch(&mut self) -> Result<usize>
```

**Example:**

```rust
// During idle time, migrate a batch
let migrated = coll.migrate_batch()?;
println!("Migrated {} vectors", migrated);
```

## HNSW Configuration

For collections under 100K vectors, HNSW provides fast in-memory search.

### Performance Characteristics

- **Construction**: O(N log N) with randomized level selection
- **Search**: O(log N) on average
- **Memory**: ~1.5KB per vector (for 384-dim embeddings)

## Vamana Configuration

For collections over 100K vectors, Vamana+PQ provides memory-efficient search.

### Parameters

```rust
const VAMANA_THRESHOLD: usize = 100_000;  // Auto-switch threshold
const DEFAULT_PQ_SUBSPACES: usize = 8;    // Subspaces for PQ
const DEFAULT_PQ_CENTROIDS: usize = 256;  // Centroids per subspace
```

### Asymmetric Distance Computation (ADC)

Vamana uses ADC for efficient search:

1. **Precompute** distance table: O(k × D)
2. **Compute** approximate distances: O(M) per vector
3. **Rerank** top candidates with exact distances

**Example internal flow:**

```rust
// Precompute distance table
let table = pq.compute_distance_table(&query);  // O(k × D)

// Score all vectors using ADC
let mut candidates: Vec<(usize, f32)> = pq_codes
    .iter()
    .enumerate()
    .map(|(i, code)| (i, pq.asymmetric_distance(&table, code)))  // O(M)
    .collect();

// Rerank top candidates
candidates.sort();
let top_k = rerank_with_exact_distance(&candidates[..rerank_count]);
```

## Error Handling

```rust
pub enum ClientError {
    Validation(String),  // Dimension mismatch, etc.
    Storage(String),     // I/O errors
    NotFound(String),    // Collection not found
}
```

**Example:**

```rust
match coll.search(&query, 10) {
    Ok(results) => {
        for r in results {
            println!("{}: {}", r.id, r.distance);
        }
    }
    Err(ClientError::Validation(msg)) => {
        eprintln!("Validation error: {}", msg);
    }
    Err(e) => {
        eprintln!("Error: {:?}", e);
    }
}
```
