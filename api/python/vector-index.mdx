The `HnswIndex` class provides approximate nearest neighbor search using Hierarchical Navigable Small World graphs. It offers ~250x speedup over brute-force search with >99% recall.

## Creating an Index

<ParamField path="dimension" type="int" required>
  Vector dimension (e.g., 768 for text embeddings, 1536 for OpenAI embeddings)
</ParamField>

<ParamField path="m" type="int" default="16">
  Maximum connections per node. Higher values improve recall but use more memory.
  - `m=16` - Balanced (recommended)
  - `m=32` - Higher recall, 2x memory
  - `m=8` - Lower memory, slightly lower recall
</ParamField>

<ParamField path="ef_construction" type="int" default="100">
  Construction search depth. Higher values improve index quality but slow down builds.
  - `ef_construction=100` - Balanced
  - `ef_construction=200` - Higher quality
  - `ef_construction=48` - Faster builds
</ParamField>

<ParamField path="metric" type="str" default="cosine">
  Distance metric:
  - `"cosine"` - Cosine similarity (normalized vectors)
  - `"euclidean"` or `"l2"` - Euclidean distance
  - `"dot"` or `"dot_product"` - Inner product
</ParamField>

<ParamField path="precision" type="str" default="f32">
  Quantization precision:
  - `"f32"` - Full precision (4 bytes/dimension)
  - `"f16"` - Half precision (2 bytes/dimension)
  - `"bf16"` - BFloat16 (2 bytes/dimension)
</ParamField>

<CodeGroup>
```python Basic
import numpy as np
from sochdb import HnswIndex

# Create index for 768D embeddings
index = HnswIndex(dimension=768)
```

```python Custom Config
from sochdb import HnswIndex

# High-quality index with custom settings
index = HnswIndex(
    dimension=1536,
    m=32,
    ef_construction=200,
    metric="cosine",
    precision="f32"
)
```
</CodeGroup>

## Inserting Vectors

### insert_batch()

Insert multiple vectors with auto-generated IDs. Uses zero-copy NumPy access and releases the GIL during HNSW construction.

<ParamField path="vectors" type="numpy.ndarray" required>
  2D float32 array of shape `(N, dimension)`. Must be C-contiguous.
</ParamField>

<ResponseField name="return" type="int">
  Number of vectors inserted
</ResponseField>

```python
import numpy as np

# Generate random embeddings
embeddings = np.random.randn(10000, 768).astype(np.float32)

# Insert (IDs are auto-generated: 0, 1, 2, ...)
count = index.insert_batch(embeddings)
print(f"Inserted {count} vectors")
```

<Note>
  **Performance**: ~15,000 vec/s for 768D vectors (10x faster than subprocess approaches)
</Note>

### insert_batch_with_ids()

Insert vectors with explicit IDs.

<ParamField path="ids" type="numpy.ndarray" required>
  1D uint64 array of IDs. Must have same length as number of vectors.
</ParamField>

<ParamField path="vectors" type="numpy.ndarray" required>
  2D float32 array of shape `(N, dimension)`. Must be C-contiguous.
</ParamField>

<ResponseField name="return" type="int">
  Number of vectors inserted
</ResponseField>

```python
# Custom IDs (e.g., from database primary keys)
ids = np.array([100, 101, 102], dtype=np.uint64)
vecs = np.random.randn(3, 768).astype(np.float32)

index.insert_batch_with_ids(ids, vecs)
```

## Searching

### search()

Find k nearest neighbors for a single query.

<ParamField path="query" type="numpy.ndarray" required>
  1D float32 array of dimension `D`
</ParamField>

<ParamField path="k" type="int" required>
  Number of neighbors to return
</ParamField>

<ParamField path="ef_search" type="int" default="k * 2">
  Search depth. Higher values improve recall but slow down search.
</ParamField>

<ResponseField name="return" type="Tuple[numpy.ndarray, numpy.ndarray]">
  Tuple of `(ids, distances)` as 1D numpy arrays:
  - `ids`: uint64 array of vector IDs
  - `distances`: float32 array of distances
</ResponseField>

```python
# Search for 10 nearest neighbors
query = np.random.randn(768).astype(np.float32)
ids, distances = index.search(query, k=10)

for i, (id, dist) in enumerate(zip(ids, distances)):
    print(f"{i+1}. ID {id}: distance {dist:.4f}")
```

### search_batch()

Search multiple queries in parallel.

<ParamField path="queries" type="numpy.ndarray" required>
  2D float32 array of shape `(Q, dimension)` for Q queries
</ParamField>

<ParamField path="k" type="int" required>
  Number of neighbors per query
</ParamField>

<ParamField path="ef_search" type="int" default="k * 2">
  Search depth
</ParamField>

<ResponseField name="return" type="Tuple[numpy.ndarray, numpy.ndarray]">
  Tuple of `(ids, distances)` as 2D numpy arrays of shape `(Q, k)`:
  - `ids[i]`: IDs for query i
  - `distances[i]`: Distances for query i
</ResponseField>

```python
# Batch search for 100 queries
queries = np.random.randn(100, 768).astype(np.float32)
ids, distances = index.search_batch(queries, k=10)

# Results for first query
print(f"Query 0 results: {ids[0]}")
print(f"Query 0 distances: {distances[0]}")
```

## Convenience Functions

### build_index()

Build an index from embeddings in one call.

<ParamField path="embeddings" type="numpy.ndarray" required>
  2D float32 array of shape `(N, D)`
</ParamField>

<ParamField path="m" type="int" default="16">
  HNSW max connections
</ParamField>

<ParamField path="ef_construction" type="int" default="100">
  Construction depth
</ParamField>

<ParamField path="metric" type="str" default="cosine">
  Distance metric
</ParamField>

<ParamField path="ids" type="numpy.ndarray" default="None">
  Optional 1D uint64 array of IDs
</ParamField>

<ResponseField name="return" type="HnswIndex">
  Index with inserted vectors
</ResponseField>

```python

```

## BatchAccumulator

For maximum insert performance, use `BatchAccumulator` to defer HNSW construction until all vectors are accumulated.

<Note>
  **Performance**: 4-5x faster inserts via zero-FFI numpy accumulation + single bulk Rayon-parallel HNSW build
</Note>

```python
from sochdb import VectorIndex

index = VectorIndex(dimension=1536, max_connections=16, ef_construction=200)

# Deferred insert: zero FFI, pure numpy memcpy
with index.batch_accumulator(estimated_size=50_000) as acc:
    for batch_ids, batch_vecs in data_loader:
        acc.add(batch_ids, batch_vecs)  # Fast accumulation
    # flush() called automatically → single bulk HNSW build

# Index is ready for search
results = index.search(query, k=10)
```

**How it works:**
- Phase 1 (Accumulate): Pure NumPy memcpy, zero FFI calls (~0.05s for 50K vectors)
- Phase 2 (Flush): Single `insert_batch()` with full Rayon parallel HNSW build (~13.7s for 50K vectors)

## Complete Example

```python
import numpy as np
from sochdb import HnswIndex

# Create index
index = HnswIndex(
    dimension=768,
    m=16,
    ef_construction=100,
    metric="cosine"
)

# Generate and insert embeddings
embeddings = np.random.randn(10000, 768).astype(np.float32)
# Normalize for cosine similarity
embeddings /= np.linalg.norm(embeddings, axis=1, keepdims=True)

count = index.insert_batch(embeddings)
print(f"Inserted {count} vectors")

# Search
query = np.random.randn(768).astype(np.float32)
query /= np.linalg.norm(query)

ids, distances = index.search(query, k=10)

print("Top 10 results:")
for rank, (id, dist) in enumerate(zip(ids, distances), 1):
    print(f"{rank}. Vector {id}: distance {dist:.4f}")

# Save for later use
index.save("embeddings.hnsw")
print(f"Saved index with {index.len} vectors")
```

## Performance Tips

1. **Use contiguous arrays**: Ensure `vectors.flags.c_contiguous == True`
   ```python
   vectors = np.ascontiguousarray(vectors)
   ```

2. **Correct dtype**: Always use `float32` for vectors and `uint64` for IDs
   ```python
   vectors = vectors.astype(np.float32)
   ids = ids.astype(np.uint64)
   ```

3. **Normalize for cosine**: Pre-normalize vectors when using cosine metric
   ```python
   vectors /= np.linalg.norm(vectors, axis=1, keepdims=True)
   ```

4. **Batch operations**: Insert in batches of 1000-10000 for best performance

5. **Use BatchAccumulator**: For large datasets (>10K vectors), use `batch_accumulator()` for 4-5x faster inserts
